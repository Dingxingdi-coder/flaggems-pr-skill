### 追加模式

**先检查 upstream 是否已有同名文件：**
```bash
git show upstream/master:tests/test_<op>.py 2>/dev/null
git show upstream/master:benchmark/test_<op>.py 2>/dev/null
```

如已存在 → 保留原文件所有代码，新测试追加到末尾。详见 `advanced-rules.md` §6。

### 六文件详细模板

#### 2.1 Kernel: `src/flag_gems/ops/<op>.py`
- 确保首行 `# Generated by KernelGen: https://github.com/flagos-ai/KernelGen`
- Review: logging（非 print）、无重复函数、无多余 import torch、无意义封装删除、hardcode size 加注释

#### 2.2 Register: `src/flag_gems/ops/__init__.py`
- 按字母序加 import + `__all__` 条目（下划线前缀模块排前面）

#### 2.3 Register: `src/flag_gems/__init__.py`
- 按字母序加 `_FULL_CONFIG` 条目：`("<aten_op>", <func>)`
- overload: `("<op>.Tensor", <func>)`; inplace: `("<op>_", <func>_)`

#### 2.4 Test: `tests/test_<op>.py`
**从 worktree 提取，不允许重写测试逻辑。** 仅允许改动：
- import 改为 `from . import accuracy_utils as utils`
- 函数名 `test_accuracy_<op>` → `test_<op>`
- 去掉 vendor-specific 分支
- 去掉 QUICK_MODE 分支

如 worktree 无测试，使用模板：
```python
import pytest
import torch
import flag_gems
from . import accuracy_utils as utils

@pytest.mark.<op>
@pytest.mark.parametrize("shape", utils.POINTWISE_SHAPES)
@pytest.mark.parametrize("dtype", utils.FLOAT_DTYPES)
def test_<op>(shape, dtype):
    inp = torch.randn(shape, dtype=dtype, device=flag_gems.device)
    ref_inp = utils.to_reference(inp)
    ref_out = torch.<op>(ref_inp)
    with flag_gems.use_gems():
        res_out = torch.<op>(inp)
    utils.gems_assert_close(res_out, ref_out, dtype)
```

#### 2.5 Benchmark: `benchmark/test_<op>.py`

| 类型 | Class |
|------|-------|
| 一元 pointwise | `base.UnaryPointwiseBenchmark` |
| 二元 pointwise | `base.BinaryPointwiseBenchmark` |
| Reduction | `base.UnaryReductionBenchmark` |
| BLAS | `base.BlasBenchmark` |
| 自定义输入 | **从 worktree 提取** Benchmark 子类（覆盖 `set_shapes`） |

模板：
```python
import pytest
import torch
from . import base, consts

@pytest.mark.<op>
def test_<op>():
    bench = base.UnaryPointwiseBenchmark(
        op_name="<op>",
        torch_op=torch.<op>,
        dtypes=consts.FLOAT_DTYPES,
    )
    bench.run()
```

#### 2.6 YAML: `conf/operators.yaml`
按 `id` 字母序插入：
```yaml
- id: <op>
  description: |
    <one-line from PyTorch docs>
  for:
    - <aten_op_name>
  labels:
    - aten
    - KernelGen
    - pointwise
  kind:
    - Math
  stages:
    - alpha: '5.1'
```

## Phase 5: Commit & Push

- **NEVER `git add -A` or `git add .`** — 687 worktrees 会被误加
- **NEVER Co-Authored-By** — CLA CI 会失败
- 只 stage 6 个文件
- 分支名 `pr/<op>`
- push 到 `fork`（Dingxingdi/FlagGems）

```bash

        tests/test_<op>.py benchmark/test_<op>.py conf/operators.yaml

```
