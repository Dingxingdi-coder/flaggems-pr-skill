# FlagGems 算子 PR 提交检查清单

每个算子 PR 提交前逐项确认。未通过任一必选项，不创建 PR。

## 0. Worktree 质量门

- [ ] 已阅读 `.worktrees/gen-<op>` 中 kernel、test、benchmark、yaml、注册文件。
- [ ] 已按 `operator-quality.md` 修复生成代码问题；PR 分支实现来自修复后的 worktree。
- [ ] kernel 核心计算不是 torch fallback。
- [ ] dtype 支持范围已用 PyTorch reference 预检测；有限 dtype 有注释和 wrapper 检查。
- [ ] 非 pointwise/custom benchmark 的 shape 逻辑在 `--level core` 下有效，非 0 cases。
- [ ] torch/gems benchmark 计算量公平。

## 1. 文件完整性

- [ ] `src/flag_gems/ops/<module>.py` — kernel 实现文件。
- [ ] `src/flag_gems/ops/__init__.py` — import 行 + `__all__` 条目，按字母序。
- [ ] `src/flag_gems/__init__.py` — `_FULL_CONFIG` 条目，按字母序。
- [ ] `tests/test_<op_id>.py` — accuracy test。
- [ ] `benchmark/test_<op_id>.py` — benchmark。
- [ ] `conf/operators.yaml` — operator entry，按 `id` 字母序。

## 2. Kernel 规范

- [ ] 首行是 `# Generated by KernelGen: https://github.com/flagos-ai/KernelGen`。
- [ ] 使用 `logging`，无 `print()`。
- [ ] 无重复函数、死代码、未使用 `@use_tl_extra` 桩函数。
- [ ] 无多余 `import torch`。
- [ ] 硬编码超参有注释；可外置配置已外置。
- [ ] 无平台 hardcode，如不必要的 `is_cuda` 分支。
- [ ] 有限 dtype 的 wrapper 有 assert/raise。

## 3. 注册规范

- [ ] `ops/__init__.py` import 与 `__all__` 包含目标 wrapper，不导出 Triton kernel 函数。
- [ ] `_FULL_CONFIG` 使用真实 aten name；overload 保留点号；inplace 保留尾部 `_`。
- [ ] 前导下划线规则符合 `naming.md`。
- [ ] 多重载时 `__all__`、`_FULL_CONFIG`、yaml、test、benchmark 数量和命名一致。

## 4. operators.yaml

- [ ] `id` 与 pytest mark、benchmark `op_name` 严格一致。
- [ ] `description` 准确。
- [ ] `for` 列出对应 aten op name。
- [ ] `labels` 包含 `aten` 和 `KernelGen`，并按类型补充分类。
- [ ] `kind` 为项目已有类别。
- [ ] 新增算子 stage 使用 `alpha: '5.1'`。

## 5. Accuracy test

- [ ] 使用 `from . import accuracy_utils as utils`。
- [ ] 测试函数名为 `test_<op_id>` 或每个 overload 一个 `test_<id>`。
- [ ] 使用 `utils.to_reference()` 构造 reference。
- [ ] 浮点用 `utils.gems_assert_close`，不传 `rtol`；精确比较用 `utils.gems_assert_equal`。
- [ ] dtype 和 shape 使用公共常量；硬编码有注释。
- [ ] 无 `print()`、QUICK_MODE、vendor-specific 分支。
- [ ] 概率算子使用统计验证。
- [ ] 不修改上游已有测试函数。

## 6. Benchmark

- [ ] 使用 pytest + `benchmark/base.py` 封装，不用自定义框架。
- [ ] pytest mark 与 `op_name` 等于 yaml `id`。
- [ ] dtype 默认用 `consts.FLOAT_DTYPES`；有限 dtype 有注释。
- [ ] 非 pointwise/custom 输入 benchmark 在 `--level core` 下非 0 cases。
- [ ] torch/gems op 参数和计算量一致。
- [ ] backward benchmark 边界公平。
- [ ] 输出 speedup/case 数已记录到 PR 数据。

## 7. 本地验证

- [ ] worktree 精度测试通过。
- [ ] worktree benchmark 通过且非 0 cases。
- [ ] `submit_operator.py` 中 `check_operator.py --strict` 通过。
- [ ] `check_overload_consistency.py` 通过。
- [ ] pre-commit 通过。
- [ ] PR 分支 pytest 通过。
- [ ] PR 分支 benchmark `--level core -s` 通过。
- [ ] 平均 speedup >= 0.8；否则已优化或放弃。

## 8. Git 与 PR

- [ ] 上游无同名 kernel/yaml id。
- [ ] 分支名为 `pr/<op>`，基于 `upstream/master`。
- [ ] `git diff --name-only upstream/master..HEAD` 只包含目标算子文件。
- [ ] 逐文件 stage，未使用 `git add -A` 或 `git add .`。
- [ ] commit message 为 `[KernelGen][Nvidia] Add <op> operator with Triton kernel`。
- [ ] commit message 无 `Co-Authored-By`。
- [ ] PR body 为英文，包含 Summary/Description、Testing/Correctness、Performance、Tested-on、Multi-backend Testing、Files Changed。
- [ ] PR 链接已回填到规范名表。
